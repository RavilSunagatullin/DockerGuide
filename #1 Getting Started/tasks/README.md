# Лабораторная работа: «Подготовь окружение Docker»

## Цель
Установить Docker и Docker Compose на своей ОС, настроить виртуализацию (WSL2 на Windows), проверить запуск контейнеров и зафиксировать вывод ключевых команд.

## Ожидаемый результат
- Docker Engine и Docker Compose установлены и работают.
- Для Windows: WSL2 активирован, выбран нужный дистрибутив.
- Сохранены выводы команд проверки и краткий отчёт по ментальной модели контейнеров.

## Структура задания
1. **Подготовка окружения.**
   - Выполни шаги установки из `theory.md` для своей ОС.
   - Windows: заскринь или запиши вывод `wsl --list --verbose`, где твой дистрибутив имеет версию 2.
   - Linux: убедись, что служба Docker активна (`systemctl status docker`).
2. **Проверка версий.**
   - Выполни `docker --version` и `docker compose version`.
   - Сохрани выводы в файлы `lab-versions.txt`.
3. **Сведения о демоне.**
   - Выполни `docker info`.
   - Сохрани вывод в `lab-info.txt`.
4. **Тестовые контейнеры.**
   - Запусти `docker run hello-world` и скопируй вывод в `lab-hello-world.txt`.
   - Запусти интерактивный контейнер `docker run -it --name alpine-lab alpine sh`, выполни внутри `cat /etc/os-release`, выйди и сохрани вывод в `lab-alpine.txt`.
5. **Управление жизненным циклом.**
   - Проверь, что контейнер остановлен: `docker ps -a | grep alpine-lab`.
   - Перезапусти: `docker start -ai alpine-lab`, снова выйди командой `exit`.
   - Удали контейнер: `docker rm alpine-lab`.
6. **Рефлексия.**
   - Ответь письменно в `lab-reflection.md`:
     - Какие компоненты устанавливает Docker Desktop/Docker Engine?
     - Чем контейнер отличается от виртуальной машины?
     - Какую роль играет Docker Compose и как проверить его наличие?

## Дополнительно (по желанию)
- Запусти `docker system df`, чтобы оценить занимаемое место.
- Если нужно очистить окружение, используй `docker system prune` (внимание: удалит неиспользуемые образы и контейнеры).

## Критерии приёмки
- Все файлы с выводами и ответами подготовлены.
- Команды `docker --version`, `docker compose version` и `docker run hello-world` выполняются без ошибок.
- Ты можешь объяснить, как включается виртуализация и зачем нужен WSL2/Hypervisor Framework.
