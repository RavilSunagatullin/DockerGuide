# Лабораторная работа: «Управление жизненным циклом контейнера»

## Цель
Научиться запускать, диагностировать и сопровождать контейнеры на основе готовых образов, используя логи, интерактивный доступ и обмен файлами.

## Ожидаемый результат
- Контейнер с Nginx запущен и доступен по опубликованному порту.
- Подготовлены примеры логов, интерактивного доступа и копирования файлов.
- Описана последовательность действий для отладки и завершения работы контейнера.

## Структура задания
1. **Запуск контейнера.**
   - Выполни `docker run --name web -d -p 8080:80 nginx`.
   - Убедись, что страница открывается на `http://localhost:8080`.
2. **Мониторинг.**
   - Посмотри активные контейнеры через `docker ps` и сохрани вывод в `lab-ps.txt`.
   - Запусти `docker logs -f web` и зафиксируй первые 20 строк в `lab-logs.txt`.
3. **Интерактивный доступ.**
   - Выполни `docker exec -it web sh`, изучи содержимое `/usr/share/nginx/html` и сохрани команды в `lab-exec.md`.
4. **Копирование файлов.**
   - Скопируй `web:/etc/nginx/nginx.conf` на хост (`lab-nginx.conf`).
   - Измени файл локально и верни обратно в контейнер под именем `default.conf`.
5. **Перезапуск и остановка.**
   - Перезапусти процесс Nginx через `docker exec web nginx -s reload`.
   - Останови и снова запусти контейнер (`docker stop web`, `docker start web`).
   - Зафиксируй наблюдения в `lab-lifecycle.md`.
6. **Очистка.**
   - Заверши работу командой `docker rm -f web`.
   - По желанию очисти остановленные контейнеры через `docker container prune` (добавь комментарий в отчёт).

## Критерии приёмки
- Все файлы отчётности и конфигурации присутствуют.
- Порт 8080 (или выбранный альтернативный) доступен в браузере во время тестирования.
- В отчёте описаны проблемы (если были) и способы диагностики.
